# Whisper Gym 設計ドキュメント

仕様の一次情報源。仕様変更はまずこのファイルを更新してから実装する。

## 1. コンセプト

ジムでのトレーニング中、iPhone に向かって**ささやき声**で
「ベンチプレス 60キロ 10回」のように話すだけで記録が残るアプリ。

- 手袋・汗・セット間の短い休憩でも、タップ数最小で記録できる
- 周囲に聞かれたくないので大声は出さない前提
- **完全オフライン・ランニングコストゼロ**(制約の詳細は CLAUDE.md)

## 2. プラットフォーム方針(決定事項)

**iPhone 向け PWA。** Safari で開き「ホーム画面に追加」してスタンドアロンで使う。

- 開発機が Windows のためネイティブ iOS 開発(Xcode/Mac 必須)は不可能、
  Apple Developer 年額 $99 も費用ゼロ制約に反する → PWA を採用(ユーザー確認済み)
- 配布: GitHub Pages(無料)。マイク使用に HTTPS が必要なため、実機確認もここで行う
- オフライン: Service Worker でアプリ本体をキャッシュ。Whisper モデルも初回 DL 後は
  端末内キャッシュから読むため、ジム(電波が悪い環境)でも動作する

## 3. 機能要件

### MVP

| ID | 機能 | 説明 |
|---|---|---|
| F1 | 音声記録 | 録音ボタン → ささやき声で発話 → 認識結果を構造化して表示 |
| F2 | 確認・修正 | 認識結果(種目/重量/回数)を保存前に確認・タップで修正できる |
| F3 | 手動記録 | 音声を使わずフォームからも入力できる(認識失敗時の逃げ道) |
| F4 | 履歴閲覧 | 日別にトレーニング内容を一覧表示 |
| F5 | 前回参照 | 記録中の種目について、前回の重量・回数を表示 |
| F6 | 種目管理 | 種目の追加・別名(認識ゆらぎ)の編集 |
| F7 | エクスポート | 全データを JSON/CSV でローカルファイルに書き出し |

### MVP に含めない

グラフ・統計、クラウド同期、App Store 配布、複数ユーザー、有酸素運動の記録(将来検討)。

## 4. 非機能要件

- 発話終了から結果表示まで **10 秒以内**(iPhone 12 相当の Safari を想定。実機計測で調整)
- 機内モードでも全機能が動作する(初回のモデルダウンロードを除く)
- 認識精度目標: 種目名+数値の抽出成功率 80% 以上(残りは F2 の修正 UI で救う)
- IndexedDB のデータはユーザーが Safari のサイトデータを消すと消える →
  エクスポート(F7)を定期的に促す表示を入れる

## 5. アーキテクチャ

```
[マイク録音]  getUserMedia + MediaRecorder (16kHz 相当に変換)
        ▼
[Whisper (ブラウザ内)]  transformers.js + ONNX モデル (WASM / WebGPU)
        │ 文字起こしテキスト: 「ベンチプレス60キロ10回」
        ▼
[パーサー (純粋関数)]   種目辞書 + 正規表現 + あいまいマッチ
        │ { exercise: "ベンチプレス", weightKg: 60, reps: 10 }
        ▼
[確認 UI] ── ユーザーが確認/修正 ──▶ [IndexedDB (Dexie)]
```

すべて端末(ブラウザ)内で完結。ネットワーク通信は初回のアプリ取得とモデルダウンロードのみ。

### レイヤー構成

- `src/speech/` — 録音制御と transformers.js 呼び出し。出力はプレーンテキスト
- `src/parser/` — テキスト → `ParsedSet` 変換。**副作用なしの純粋関数**(最重要モジュール)
- `src/db/` — Dexie アクセスの唯一の窓口。クエリ関数として公開
- `src/screens/` — 画面。上記レイヤーを組み合わせるだけ

## 6. 音声認識設計

### モデル候補(フェーズ 1 で実機比較して決定)

| モデル | サイズ(量子化後目安) | 備考 |
|---|---|---|
| onnx-community/whisper-base | 約 60MB | 多言語。第一候補 |
| onnx-community/kotoba-whisper 系 | 約 100〜150MB | 日本語特化 distil-whisper。精度期待大 |
| onnx-community/whisper-small | 約 190MB | 精度は高いが iPhone Safari のメモリと相談 |

- モデルは Hugging Face から初回に無料ダウンロードし、ブラウザのキャッシュに保存
  (transformers.js が自動でやる)。アプリ本体には同梱しない
- `language: "japanese"`、`task: "transcribe"` を固定
- 比較のベースラインとして **Web Speech API**(iOS Safari の無料音声認識)も
  フェーズ 1 で試す。ささやき声で Whisper に劣らなければ、こちらの方が軽い

### ささやき声対策

ささやき声は声帯振動がなく認識精度が落ちる。以下で補う:

1. **運用**: マイクに口を近づけて話す前提の UI(録音中は画面に「口元 10cm」ガイド表示)
2. **録音品質**: `echoCancellation`/`noiseSuppression` 有効の getUserMedia 設定
3. **後段吸収**: パーサーのあいまいマッチで誤認識を辞書に寄せる
4. **最終手段**: 確認 UI(F2)と手動入力(F3)

### 録音フロー

- ボタン押下で録音開始 → もう一度押して終了(トグル式。手袋・汗対策)
- 最大録音長 15 秒(1 発話 = 1 セットの記録を想定)
- iOS の制約: 録音開始はユーザー操作(タップ)起点でのみ可能

## 7. パーサー設計

### 入力想定(発話文法)

```
<種目名> <重量>キロ <回数>回
例: 「ベンチプレス 60キロ 10回」
    「ラットプルダウン 40キロ 12回」
    「スクワット 80キロ 8回」
    「腕立て伏せ 20回」          ← 自重種目は重量省略可
    「六十二点五キロ」のような漢数字・小数も受け付ける
```

### 処理ステップ

1. **正規化**: 全角→半角、漢数字→算用数字(「六十」→60、「点」→小数点)、空白・句読点除去
2. **数値抽出**: 重量 `(\d+(\.\d+)?)(キロ|kg|㎏)`、回数 `(\d+)(回|レップ)`。
   数値部分を除去した残りを種目名候補とする
3. **種目マッチング**(順にフォールバック):
   - 完全一致 → 別名一致 → 部分一致 → あいまいマッチ(編集距離)
   - あいまいマッチは閾値(文字数の 1/3 以内の編集距離)を下回れば「種目不明」とし、
     確認 UI で選択させる
4. **結果**: `ParsedSet { exerciseName | null, weightKg | null, reps | null, rawText, confidence }`
   - 欠損フィールドは UI 側で入力を促す。**推測で埋めない**

### 種目辞書

- 初期シード: 主要 30 種目強(BIG3、マシン系、ダンベル系、自重系)を `src/parser/exercises.ts` に持つ
- 各種目に別名リスト(例: ベンチプレス ← 「ベンチ」「ベンプレ」。
  誤認識ゆらぎ「弁地プレス」等はフェーズ 1 の実測で追加)
- ユーザーが種目・別名を追加可能(F6)。実行時の辞書は DB に持ち、初回起動時にシードを投入

## 8. データモデル(IndexedDB / Dexie)

```ts
// exercises: 種目マスタ
{ id, name /* unique */, category /* 胸/背中/脚/肩/腕/体幹 */, isBodyweight, createdAt }

// exerciseAliases: 種目の別名・認識ゆらぎ
{ id, exerciseId, alias /* unique */ }

// sets: 1 セット = 1 レコード
{ id, exerciseId, weightKg /* 自重は null */, reps,
  performedAt /* ISO8601。日別表示はこれで集計 */,
  rawText /* 認識テキスト原文。デバッグ・辞書改善用 */, createdAt }

// インデックス: exercises.name(unique), exerciseAliases.alias(unique),
//              sets.performedAt, sets.exerciseId
```

- 「ワークアウト(1回のジム訪問)」テーブルは持たない。`performedAt` の日付で
  グルーピングすれば十分(MVP の簡素化)
- `rawText` を保存しておき、誤認識パターンをパーサーのテストケースと別名辞書に還元する
- スキーマ変更は Dexie の `version(n).stores()` + `upgrade()` でマイグレーション

## 9. 画面設計

```
[記録画面]  ← ホーム。起動直後にここ
  ┌──────────────────────┐
  │ 今日: ベンチプレス 60kg×10 …│ ← 今日の記録がライブで積まれる
  │                              │
  │        ● 録音ボタン(大)     │ ← トグル式。録音中はガイド表示
  │                              │
  │ 認識結果カード               │ ← 種目/重量/回数。各項目タップで修正
  │ [保存] [破棄] [手動入力]     │
  └──────────────────────┘
      前回同種目: 57.5kg×10 (6/28) ← F5

[履歴画面]   日付ごとの一覧 → 日付タップで詳細(種目別セット一覧)
[種目管理]   種目一覧 → 追加/別名編集
[設定]       モデル管理(再DL/切替)、エクスポート
```

ナビゲーション: 下部タブ 4 つ(記録/履歴/種目/設定)。

## 10. エラー・エッジケース

- **認識結果が空/種目不明**: 認識テキスト原文を表示し、種目選択 or 手動入力へ誘導
- **モデル未ダウンロードで録音開始**: ダウンロード進捗 UI へ誘導(Wi-Fi 推奨の注意書き)
- **マイク権限拒否**: 手動入力のみで使える旨を表示
- **Safari のサイトデータ削除でデータ消失**: エクスポートを促すバナー(最終エクスポートから
  30 日超で表示)
- **同一発話に複数セット**(「60キロ10回を3セット」): MVP では非対応。
  1 発話 1 セットを UI ガイドで促す。将来パーサー拡張で対応

| 依存 | ライセンス/費用 |
|---|---|
| transformers.js(@huggingface/transformers) | Apache-2.0。ブラウザ内実行のため利用料なし |
| Whisper 系 ONNX モデル(Hugging Face 配布) | MIT / Apache-2.0。DL 無料 |

外部サーバーに依存する箇所は**静的ホスティングと初回モデルダウンロードのみ**。
万一モデル配布が止まった場合に備え、モデル ID は設定で差し替え可能にする。
