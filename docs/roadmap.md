# プロジェクトの進め方

## 基本方針

- **仕様変更・機能追加はまず `docs/design.md` を更新してから実装する**。設計と実装を乖離させない
- 各フェーズの終わりに iPhone 実機(Safari)で動作確認してから次へ進む
- 音声認識の精度問題は「モデルを大きくする」前に、録音品質の改善・パーサー側での吸収を先に検討する
- コミットは機能単位で小さく。メッセージは日本語でよい

## iPhone 実機での確認手順

マイク使用に HTTPS が必須のため、GitHub Pages にデプロイして確認する:

1. GitHub に公開リポジトリを作成し push
2. GitHub Actions で `npm run build` → `dist/` を Pages にデプロイ(ワークフロー整備はフェーズ 1)
3. iPhone の Safari で開く →「ホーム画面に追加」

PC 上の開発は `npm run dev` + ブラウザで行う(パーサー・UI・DB はここで大半を確認できる)。

## 開発フェーズ

### フェーズ 0: 環境構築

- [x] Vite + React + TypeScript(strict)プロジェクト作成
- [x] Vitest セットアップ、型チェック・ビルドが通ることを確認
- [x] git リポジトリ初期化、`.gitignore` 整備
- [x] 4 タブの画面骨格(記録/履歴/種目/設定)と Dexie スキーマ

### フェーズ 1: 技術検証(最大のリスクを先に潰す)

このアプリの成否は「**ささやき声を iPhone Safari 上の Whisper が実用精度で認識できるか**」に
かかっている。UI を作り込む前にここを検証する。

- [x] GitHub Actions デプロイワークフロー作成(`.github/workflows/deploy.yml`)
- [ ] GitHub リポジトリ作成 + Pages 自動デプロイ(push が残)
- [x] transformers.js を組み込み、whisper-tiny / base / small を切り替えられる検証画面を作る
      (kotoba-whisper 系は実サイズ判明により候補から除外。design.md §6 参照)
- [ ] iPhone 実機で、ジムを想定したささやき声の認識率・処理時間・メモリ挙動を
      **モデル × (WASM / WebGPU)** のマトリクスで計測
- [ ] 無音・小声録音での幻覚出力(「ご視聴〜」等)の頻度を確認し、RMS ゲートの閾値を決める
- [ ] 参考ベースラインとして Web Speech API も Safari タブで計測
      (スタンドアロン PWA では動作せず、オフラインでも動かないため**代替案にはならない**)
- [ ] PWA(ホーム画面追加)状態でマイクが使えることを確認
      (仮アイコン+apple-touch-icon を用意した上で検証する)
- [ ] **マイク権限プロンプトの頻度**を確認(アプリ再起動後・画面ロック復帰後に再要求されるか)
- [ ] 採用モデルを決定し、design.md に結果を記録
- 判断基準: 種目名+数値が 8 割以上拾えれば OK(細部のゆらぎはパーサーで吸収する)
- **精度が全く出ない場合**: 録音品質改善 → モデル変更 → 確認 UI・手動入力の UX 強化 →
  最後に「ささやき声」要件自体を design.md で再検討(通常声量の許容)する

### フェーズ 2: パーサー(音声テキスト → 構造化データ)

- [ ] 種目辞書の初期シード(主要 30 種目強+別名)
- [ ] 正規化(全角半角・漢数字)と数値抽出(「60キロ」「六十キロ」「10回」)
- [ ] あいまいマッチング(誤認識された種目名を辞書に寄せる)
- [ ] フェーズ 1 で採取した実際の誤認識サンプルをユニットテスト化

### フェーズ 3: データ層

- [ ] Dexie スキーマ実装(design.md のデータモデル通り)+ 初期シード投入
- [ ] CRUD クエリ関数+テスト

### フェーズ 4: MVP UI

- [ ] 記録画面: 録音ボタン → 認識結果の確認・修正 → 保存
- [ ] 手動入力フォーム(認識失敗時の逃げ道)
- [ ] 履歴画面: 日別のトレーニング一覧
- [ ] 種目管理画面: 種目追加・別名編集

### フェーズ 5: 仕上げ

- [ ] 前回記録の参照(同種目の前回重量・回数を表示)
- [ ] JSON/CSV エクスポート+ **JSON インポート(F8。復元機能)**
- [ ] モデルダウンロードの進捗 UI(Wi-Fi 推奨の注意書き)
- [ ] PWA 仕上げ(本アイコン、スプラッシュ、オフライン確認。仮アイコンはフェーズ 1 で作成済み)
- [ ] ホーム画面追加を促すオンボーディング(7 日 ITP 消去対策)
- [ ] エクスポート促しバナー(データ消失対策)

## 将来検討(MVP には含めない)

- 記録の可視化(重量の推移グラフ)
- 複数セットの一括発話(「60キロ10回を3セット」)
- 単位なし数値の発話(「ベンチプレス 60 10」。数値が 2 つ残ったら重量→回数とみなすヒューリスティック)
- 時間ベース種目(プランク等。`sets.durationSec` の追加が必要)
- ユーザー自身のクラウドストレージへのバックアップ(無料の範囲で)
- ネイティブ iOS 化(Mac / Apple Developer 契約が手に入った場合)
